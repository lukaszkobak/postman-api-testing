# Postman API Testing - Łukasz Kobak

A collection of Postman projects demonstrating API testing skills across multiple public APIs.

## What's Covered

### CRUD Operations
Full request lifecycle (GET, POST, PUT, PATCH, DELETE) tested against JSONPlaceholder.

### Array Methods & Assertions
Using `filter`, `find`, and `forEach` to validate API responses, combined with Chai assertions (`pm.expect`) across CoinGecko, GitHub, and Dog CEO APIs.

### Dynamic Data & Request Chaining
Pre-request scripts generating dynamic data (random values, timestamps), environment variable chaining between requests, and test data cleanup - demonstrated on the Simple Books API order lifecycle.

### Authentication
Bearer Token authentication, set at both request and collection level.

## About
Practical, hands-on Postman skills built through self-directed practice across real public APIs (OpenWeather, GitHub, CoinGecko, Dog CEO, JSONPlaceholder, Simple Books).

**Tools:** Postman, JavaScript (pm.test, pm.expect, Chai assertions)
**Focus:** API testing, request automation, technical support scenarios
